# Android / Web UI 视觉规范

本文档是迁移项目所有页面的 UI 统一基线。Android Jetpack Compose 是视觉基准，Web 使用现有 CSS 变量和等价组件实现；功能、数据和路由差异不应改变页面的层级、节奏和触控体验。

## 1. 页面层级

页面必须遵循以下从属关系，不允许跳级：

1. 页面：返回栏或全局顶栏，以及页面标题。
2. 一级分区：一类业务或设置的完整区域，使用独立标题和说明。
3. 卡片：承载一组相关内容，使用统一表面色和圆角。
4. 子配置：卡片内部的字段、状态或操作，以分隔线和较小字号表示。

“高级设置”“诊断”“缓存”等横向技术能力必须是独立一级分区，不得视觉上成为上一张业务卡的子项。全局设置与平台设置也必须使用不同的一级分区标题。

## 2. 字体层级

默认字体为 `Roboto, Noto Sans SC, system-ui, sans-serif`。除数据展示专用例外外，页面使用以下 token：

| 语义 | Web | Android | 字重 | 用途 |
| --- | ---: | ---: | ---: | --- |
| 页面标题 | 20px / 28px | 20sp | 700 | 二级页标题 |
| 全局主标题 | 24px / 30px | 24sp | 700 | 顶栏账本名、顶层主标题 |
| 一级分区标题 | 18px / 24px | 18sp | 700 | 通用偏好、平台配置、收益日历 |
| 卡片标题 | 16px / 22px | 16sp | 600 | 平台名、趋势卡、功能卡 |
| 子配置标题 | 14px / 20px | 14sp | 600 | 费率方案、结单密码、字段组 |
| 正文 | 13px / 19px | 13sp | 400 | 说明与常规内容 |
| 辅助说明 | 12px / 18px | 12sp | 400 | 状态、日期、帮助文本 |
| 紧凑元数据 | 10px / 14px | 10sp | 400–600 | 标签、日历收益、次要标记 |

字号必须随层级递减：子配置标题不得大于所属卡片标题，辅助说明不得大于正文。金额、收益率、资产总值等数据可以使用页面专用大数字 token，但不能用作普通标题。

## 3. 间距与尺寸

| token | 值 | 使用场景 |
| --- | ---: | --- |
| 页面横向边距 | 20px | 手机页面主体 |
| 页面安全区顶部 | 12–16px | 二级页与状态栏之间 |
| 一级分区间距 | 24px | 不同配置/业务类别之间 |
| 页面常规间距 | 16px | 页面内主要内容块 |
| 标题到内容 | 10px | 分区标题或卡片标题之后 |
| 卡片内边距 | 16px | Android `SurfaceSecondary` 卡片 |
| 卡片内分组间距 | 12px | 卡片内子配置组 |
| 列表项间距 | 8–12px | 平台、邮箱、排行列表 |
| 卡片圆角 | 16px | 主卡片和分区容器 |
| 紧凑控件圆角 | 8–12px | 输入框、按钮、标签 |
| 最小触控尺寸 | 40px | 图标按钮、主要操作 |

页面宽度采用手机优先布局，内容容器最大宽度 520px；390、414、430px 宽度必须保证文字不截断、控件可点击、底部安全区不遮挡内容。

## 4. 颜色与表面

- 浅色页面背景使用白色或 `#F7F7F8`；Android 次级表面使用 `#F7F8FA`。
- 深色次级表面使用 `#1E1E1E`，不得用半透明渐变替代 Android 表面色。
- 主文字使用 `#1A1A1A`，次级文字使用 `#4B5563`，辅助文字使用 `#9CA3AF`；对应 Web 变量为 `--text-primary`、`--text-secondary`、`--text-muted`。
- 主卡片统一使用 `--android-surface-secondary` 或等价 Android `SurfaceSecondary`，不在同一页面混用多种相近灰色。
- 涨跌颜色遵循现有市场配色设置；警告使用浅黄色表面并配细边框，错误/成功状态不得改变布局层级。
- 主题设置统一使用 `system`、`light`、`dark` 三种外观模式；显式模式覆盖系统外观，跟随系统时响应设备主题变化。

## 5. 组件规则

- 顶栏：使用矢量图标，图标按钮不使用字符箭头；标题、平台/账本副标题按两行布局。
- 二级页返回栏：必须使用 `position: sticky` 固定在内容顶部，包含安全区内边距、40px 返回按钮和 20px 页面标题；返回栏不能随主体内容滚走，右侧可放删除、刷新等页面操作。
- 二级页手势：触屏设备支持从左侧 32px 内开始、水平移动至少 72px 的右滑返回；水平位移必须明显大于垂直位移。输入控件、图表横向滚动、抽屉和对话框区域必须屏蔽该手势。
- 二级页深链：没有可用历史记录时，返回必须回到对应一级页（交易表单回流水、分析详情回分析、数据子页回数据、设置回持仓），不能停留在空白或不可返回页面。
- 卡片：标题、说明、内容按固定顺序排列；同一层级的卡片表面、圆角、内边距必须一致。
- 表单：标签在控件上方，标签字号不大于子配置标题；帮助文本位于控件下方并使用辅助字号。
- 通用偏好列表：配置名称在左、当前值和状态图标在右，行高约 64px，行间使用细分割线；不使用显式保存按钮。
- 偏好选择：点击列表行打开底部单选弹层，选中项显示勾选并立即保存；弹层必须处理遮罩、返回键和底部安全区。
- 分段筛选：选中态使用实心或高对比表面，未选中态保持弱色；控件高度和圆角统一。
- 抽屉与底部弹层：必须处理状态栏和底部安全区，内容区域可滚动，取消/确认按钮始终可见。
- 空态、同步卡和提示卡：作为独立内容块渲染，不嵌套到不相关业务卡中。

## 6. 页面审查清单

- 是否能一眼区分页面、一级分区、卡片和子配置？
- 是否所有标题按 20/18/16/14px 递减，说明统一为 13/12px？
- 是否使用统一的 20px 边距、16px 卡片、16px 圆角和 12px 内部分组间距？
- 二级页滚动到任意位置时，返回按钮是否仍在顶部可见？从左边缘右滑是否能返回，且不会干扰表单、图表和弹层？
- 是否存在字符箭头、超过父级字号的子标题或混用表面色？
- 390/414/430px 下是否无截断、重叠、不可点击和安全区遮挡？
- 浅色/深色、空数据、错误和加载状态是否保持同一层级结构？

后续页面 UI 审查应在对应映射条目中引用本文档；若 Android 页面存在专用数据展示字号，应在页面实现或快照说明中明确标注例外。
